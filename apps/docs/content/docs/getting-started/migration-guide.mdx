This guide helps you migrate from popular UI libraries and design systems to Bank Kit UI, highlighting key differences and providing migration strategies.

## From Shadcn/ui

Bank Kit UI is built on top of Shadcn/ui, so migration is straightforward for most components.

### Component Mapping

Most Shadcn/ui components have direct equivalents in Bank Kit UI:

| Shadcn/ui | Bank Kit UI | Notes |
|-----------|-------------|--------|
| `@/components/ui/button` | `@bank-kit/ui/components/button` | Same API |
| `@/components/ui/card` | `@bank-kit/ui/components/card` | Same API |
| `@/components/ui/input` | `@bank-kit/ui/components/input` | Enhanced with financial validation |
| `@/components/ui/dialog` | `@bank-kit/ui/components/dialog` | Same API |

### Migration Steps

1. **Update Imports**:
```tsx
// Before (Shadcn/ui)
import { Button } from "@/components/ui/button";
import { Card, CardContent } from "@/components/ui/card";

// After (Bank Kit UI)
import { Button } from "@bank-kit/ui/components/button";
import { Card, CardContent } from "@bank-kit/ui/components/card";
```

2. **Update Global Styles**:
```css
/* Before */
@import "./globals.css";

/* After */
@import "@bank-kit/ui/globals.css";
```

3. **Configuration**:
```json title="components.json"
{
  "style": "default",
  "rsc": false,
  "tsx": true,
  "tailwind": {
    "config": "tailwind.config.js",
    "css": "app/globals.css",
    "baseColor": "slate",
    "cssVariables": true
  },
  "aliases": {
    "components": "@bank-kit/ui/components",
    "utils": "@bank-kit/ui/lib/utils"
  }
}
```

### Enhanced Components

Bank Kit UI adds financial-specific enhancements:

```tsx
// Before (Basic Shadcn/ui input)
<Input type="text" placeholder="Amount" />

// After (Bank Kit UI with currency formatting)
<CurrencyInput
  currency="USD"
  placeholder="Amount"
  onValueChange={(value) => console.log(value)}
/>
```

## From Material-UI (MUI)

### Component Mapping

| Material-UI | Bank Kit UI | Migration Notes |
|-------------|-------------|-----------------|
| `Button` | `Button` | Update variant prop values |
| `Card` | `Card` | Restructure content components |
| `TextField` | `Input` | Different API structure |
| `Dialog` | `Dialog` | Similar API |
| `AppBar` | Custom component | Build with Bank Kit primitives |

### Migration Steps

1. **Button Migration**:
```tsx
// Before (MUI)
import { Button } from '@mui/material';

<Button variant="contained" color="primary">
  Submit
</Button>

// After (Bank Kit UI)
import { Button } from '@bank-kit/ui/components/button';

<Button variant="default">
  Submit
</Button>
```

2. **Input Migration**:
```tsx
// Before (MUI)
import { TextField } from '@mui/material';

<TextField
  label="Email"
  variant="outlined"
  fullWidth
/>

// After (Bank Kit UI)
import { Input } from '@bank-kit/ui/components/input';
import { Label } from '@bank-kit/ui/components/label';

<div className="w-full">
  <Label htmlFor="email">Email</Label>
  <Input id="email" type="email" className="w-full" />
</div>
```

3. **Card Migration**:
```tsx
// Before (MUI)
import { Card, CardContent, CardActions } from '@mui/material';

<Card>
  <CardContent>
    <Typography variant="h5">Title</Typography>
    <Typography variant="body2">Content</Typography>
  </CardContent>
  <CardActions>
    <Button>Action</Button>
  </CardActions>
</Card>

// After (Bank Kit UI)
import { Card, CardContent, CardFooter, CardHeader, CardTitle } from '@bank-kit/ui/components/card';

<Card>
  <CardHeader>
    <CardTitle>Title</CardTitle>
  </CardHeader>
  <CardContent>
    <p>Content</p>
  </CardContent>
  <CardFooter>
    <Button>Action</Button>
  </CardFooter>
</Card>
```

### Theming Migration

```tsx
// Before (MUI Theme)
import { createTheme, ThemeProvider } from '@mui/material/styles';

const theme = createTheme({
  palette: {
    primary: {
      main: '#1976d2',
    },
  },
});

// After (Bank Kit UI with CSS Variables)
// Configure in your CSS
:root {
  --primary: 221.2 83.2% 53.3%;
  --primary-foreground: 210 40% 98%;
}
```

## From Ant Design

### Component Mapping

| Ant Design | Bank Kit UI | Migration Notes |
|------------|-------------|-----------------|
| `Button` | `Button` | Different size props |
| `Card` | `Card` | Restructure with Header/Content/Footer |
| `Input` | `Input` | Different validation approach |
| `Form` | Custom form patterns | Use with react-hook-form |
| `Table` | `Table` | Different data structure |

### Migration Steps

1. **Form Migration**:
```tsx
// Before (Ant Design)
import { Form, Input, Button } from 'antd';

<Form onFinish={onFinish}>
  <Form.Item name="username" rules={[{ required: true }]}>
    <Input placeholder="Username" />
  </Form.Item>
  <Form.Item>
    <Button type="primary" htmlType="submit">
      Submit
    </Button>
  </Form.Item>
</Form>

// After (Bank Kit UI with react-hook-form)
import { useForm } from 'react-hook-form';
import { Input } from '@bank-kit/ui/components/input';
import { Button } from '@bank-kit/ui/components/button';
import { Label } from '@bank-kit/ui/components/label';

const { register, handleSubmit } = useForm();

<form onSubmit={handleSubmit(onSubmit)}>
  <div>
    <Label htmlFor="username">Username</Label>
    <Input 
      id="username"
      {...register('username', { required: true })}
      placeholder="Username" 
    />
  </div>
  <Button type="submit">Submit</Button>
</form>
```

2. **Table Migration**:
```tsx
// Before (Ant Design)
import { Table } from 'antd';

const columns = [
  { title: 'Name', dataIndex: 'name', key: 'name' },
  { title: 'Age', dataIndex: 'age', key: 'age' },
];

<Table dataSource={data} columns={columns} />

// After (Bank Kit UI)
import { Table, TableBody, TableCell, TableHead, TableHeader, TableRow } from '@bank-kit/ui/components/table';

<Table>
  <TableHeader>
    <TableRow>
      <TableHead>Name</TableHead>
      <TableHead>Age</TableHead>
    </TableRow>
  </TableHeader>
  <TableBody>
    {data.map((item) => (
      <TableRow key={item.id}>
        <TableCell>{item.name}</TableCell>
        <TableCell>{item.age}</TableCell>
      </TableRow>
    ))}
  </TableBody>
</Table>
```

## From Chakra UI

### Component Mapping

| Chakra UI | Bank Kit UI | Migration Notes |
|-----------|-------------|-----------------|
| `Button` | `Button` | Different color scheme props |
| `Box` | `div` with Tailwind classes | Use Tailwind utilities |
| `Input` | `Input` | Different focus management |
| `Modal` | `Dialog` | Different API structure |
| `Flex` | `div` with flex classes | Use Tailwind flex utilities |

### Migration Steps

1. **Layout Migration**:
```tsx
// Before (Chakra UI)
import { Box, Flex, Spacer } from '@chakra-ui/react';

<Flex>
  <Box p="4">
    <Text>Left</Text>
  </Box>
  <Spacer />
  <Box p="4">
    <Text>Right</Text>
  </Box>
</Flex>

// After (Bank Kit UI with Tailwind)
<div className="flex">
  <div className="p-4">
    <p>Left</p>
  </div>
  <div className="flex-1"></div>
  <div className="p-4">
    <p>Right</p>
  </div>
</div>
```

2. **Button Migration**:
```tsx
// Before (Chakra UI)
import { Button } from '@chakra-ui/react';

<Button colorScheme="blue" size="lg">
  Click me
</Button>

// After (Bank Kit UI)
import { Button } from '@bank-kit/ui/components/button';

<Button variant="default" size="lg">
  Click me
</Button>
```

## From Bootstrap/React-Bootstrap

### Component Mapping

| Bootstrap | Bank Kit UI | Migration Notes |
|-----------|-------------|-----------------|
| `Button` | `Button` | Different variant names |
| `Card` | `Card` | More structured API |
| `Form.Control` | `Input` | Different validation |
| `Modal` | `Dialog` | Different API |
| `Nav` | Custom navigation | Build with primitives |

### Migration Steps

1. **Grid System Migration**:
```tsx
// Before (Bootstrap)
import { Container, Row, Col } from 'react-bootstrap';

<Container>
  <Row>
    <Col md={6}>Content 1</Col>
    <Col md={6}>Content 2</Col>
  </Row>
</Container>

// After (Bank Kit UI with Tailwind)
<div className="container mx-auto">
  <div className="grid grid-cols-1 md:grid-cols-2 gap-4">
    <div>Content 1</div>
    <div>Content 2</div>
  </div>
</div>
```

2. **Form Migration**:
```tsx
// Before (Bootstrap)
import { Form, Button } from 'react-bootstrap';

<Form>
  <Form.Group>
    <Form.Label>Email</Form.Label>
    <Form.Control type="email" placeholder="Enter email" />
  </Form.Group>
  <Button variant="primary" type="submit">
    Submit
  </Button>
</Form>

// After (Bank Kit UI)
import { Input } from '@bank-kit/ui/components/input';
import { Button } from '@bank-kit/ui/components/button';
import { Label } from '@bank-kit/ui/components/label';

<form>
  <div className="mb-4">
    <Label htmlFor="email">Email</Label>
    <Input id="email" type="email" placeholder="Enter email" />
  </div>
  <Button type="submit">Submit</Button>
</form>
```

## Migration Strategies

### Incremental Migration

1. **Start with New Features**: Use Bank Kit UI for new components
2. **Replace Leaf Components**: Migrate buttons, inputs first
3. **Update Layout Components**: Cards, modals, forms
4. **Migrate Complex Components**: Tables, navigation

### Testing Strategy

```tsx
// Create component adapters during migration
import { Button as MUIButton } from '@mui/material';
import { Button as BankKitButton } from '@bank-kit/ui/components/button';

interface ButtonProps {
  variant?: 'primary' | 'secondary';
  children: React.ReactNode;
  onClick?: () => void;
}

export function Button({ variant = 'primary', ...props }: ButtonProps) {
  // Feature flag to switch between implementations
  if (process.env.USE_BANK_KIT === 'true') {
    return (
      <BankKitButton 
        variant={variant === 'primary' ? 'default' : 'secondary'} 
        {...props} 
      />
    );
  }
  
  return (
    <MUIButton 
      variant={variant === 'primary' ? 'contained' : 'outlined'} 
      {...props} 
    />
  );
}
```

## Common Migration Issues

### Styling Conflicts

```css
/* Reset conflicting styles */
.migration-container {
  /* Remove existing component library styles */
  all: unset;
}

.migration-container * {
  /* Apply Bank Kit UI styles */
  @apply border-border bg-background text-foreground;
}
```

### TypeScript Issues

```tsx
// Create type adapters for gradual migration
type LegacyButtonProps = {
  color: 'primary' | 'secondary';
  size: 'small' | 'medium' | 'large';
};

type BankKitButtonProps = {
  variant: 'default' | 'secondary';
  size: 'sm' | 'default' | 'lg';
};

function adaptButtonProps(props: LegacyButtonProps): BankKitButtonProps {
  return {
    variant: props.color === 'primary' ? 'default' : 'secondary',
    size: props.size === 'small' ? 'sm' : props.size === 'large' ? 'lg' : 'default',
  };
}
```

### Performance Optimization

```tsx
// Lazy load Bank Kit components during migration
import { lazy, Suspense } from 'react';

const BankKitTable = lazy(() => import('@bank-kit/ui/components/table'));

function DataTable({ useBankKit, ...props }) {
  if (useBankKit) {
    return (
      <Suspense fallback={<div>Loading...</div>}>
        <BankKitTable {...props} />
      </Suspense>
    );
  }
  
  return <LegacyTable {...props} />;
}
```
