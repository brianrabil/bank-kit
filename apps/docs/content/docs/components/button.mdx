## Overview

The **Button** is a fundamental UI element in bank‑kit.
It supports multiple **variants**, **sizes**, icon placement, and built‑in loading states.

<div className="mt-6 flex flex-wrap gap-4">
  <Button>Transfer Funds</Button>
  <Button variant="secondary">View Statement</Button>
  <Button variant="outline">Settings</Button>
  <Button variant="destructive">Close Account</Button>
</div>

## Installation

```bash
npm i @bank-kit/ui
```

```tsx
import { Button } from '@bank-kit/ui';
```

## Accessibility

- Uses semantic `<button>` elements.
- Keyboard‑navigable & focus‑visible.
- ARIA attributes applied during loading.
- Color contrast meets WCAG AA.

<Callout className="mt-4">
  For icon‑only buttons, include `aria‑label` or visually hidden text.
</Callout>

## Customization

Tailwind utility classes let you theme buttons quickly:

```tsx
<Button
  variant="outline"
  className="border-emerald-500 text-emerald-600 hover:bg-emerald-50 hover:text-emerald-700"
>
  Custom Style
</Button>
```

Need deeper changes? Extend the `buttonVariants` helper and roll your own:

```tsx
import { buttonVariants } from '@bank-kit/ui';
import { cn } from '@/lib/utils';

export function GradientButton(props) {
  return (
    <button
      className={cn(
        buttonVariants({ variant: 'primary' }),
        'bg-gradient-to-r from-blue-500 to-purple-500 hover:from-blue-600 hover:to-purple-600'
      )}
      {...props}
    />
  );
}
```
