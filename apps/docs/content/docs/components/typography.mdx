# Typography

Bank Kit provides a unified typography system to keep your text styles consistent across all financial components and flows. This guide covers our **Display** (large headings) and **Text** (body) scales, plus recommended font weights, line heights, and usage patterns.

---

## Overview

Bank Kit’s typography system comprises **six Display sizes** and **five Text sizes**, each available in four weights:

- **Regular** (400)
- **Medium** (500)
- **Semibold** (600)
- **Bold** (700)

### Display Sizes
- **Display 2xl** – Typically used for hero headlines or marketing banners.
- **Display xl** – Main section headings or high-level titles.
- **Display lg** – Secondary headings or prominent page titles.
- **Display md** – Tertiary headings or sub-sections.
- **Display sm** – Smaller display headings or side-section titles.
- **Display xs** – Smallest display headings, often used as subtitles.

### Body Text Sizes
- **Text xl** – Larger body text or introductory paragraphs.
- **Text lg** – Standard paragraphs when readability and hierarchy matter.
- **Text md** – Default body copy for most use cases.
- **Text sm** – Smaller text for secondary info (labels, metadata).
- **Text xs** – Smallest text, typically for disclaimers, fine print, or footnotes.

## Tailwind Configuration

If you are using Tailwind CSS, you can define these type scales in your `tailwind.config.js`. Here’s an example snippet that maps each label to its font size, line height, and letter spacing:

```ts
// tailwind.config.ts
module.exports = {
  theme: {
    extend: {
      fontSize: {
        // Display sizes
        'display-2xl': ['60px', {
          lineHeight: '72px',
          letterSpacing: '-2%',
        }],
        'display-xl': ['48px', {
          lineHeight: '60px',
          letterSpacing: '-2%',
        }],
        'display-lg': ['32px', {
          lineHeight: '44px',
          letterSpacing: '-2%',
        }],
        'display-md': ['30px', {
          lineHeight: '38px',
          letterSpacing: '-2%',
        }],
        'display-sm': ['24px', {
          lineHeight: '32px',
          letterSpacing: '-2%',
        }],
        'display-xs': ['20px', {
          lineHeight: '28px',
          letterSpacing: '-1.5%',
        }],

        // Text sizes
        'text-xl': ['18px', {
          lineHeight: '28px',
        }],
        'text-lg': ['16px', {
          lineHeight: '24px',
        }],
        'text-md': ['14px', {
          lineHeight: '20px',
        }],
        'text-sm': ['12px', {
          lineHeight: '18px',
        }],
        'text-xs': ['10px', {
          lineHeight: '16px',
        }],
      },
      // You can also define weights in your tailwind classes
      // or simply use font-[weight] in your HTML/JSX.
    }
  }
}
```

After adding these to your config, you can use classes like text-display-2xl font-bold or text-md font-semibold throughout your app.

## Usage Examples

### Display Headings

```tsx
import React from "react"

export function DisplayExamples() {
  return (
    <div className="space-y-4">
      <h1 className="text-display-2xl font-regular">
        Display 2xl Regular
      </h1>
      <h1 className="text-display-2xl font-medium">
        Display 2xl Medium
      </h1>
      <h1 className="text-display-2xl font-semibold">
        Display 2xl Semibold
      </h1>
      <h1 className="text-display-2xl font-bold">
        Display 2xl Bold
      </h1>
    </div>
  )
}
```

Tip: Use Display styles sparingly. They’re meant to make a bold statement—ideal for landing pages, dashboards, or major headings.

### Body Text

```tsx
import React from "react"

export function TextExamples() {
  return (
    <div className="space-y-2">
      <p className="text-text-md font-regular">
        Text md Regular – This is the standard body text size used in most paragraphs.
      </p>
      <p className="text-text-md font-semibold">
        Text md Semibold – Use semibold for inline emphasis or short highlights.
      </p>
      <p className="text-text-sm font-medium">
        Text sm Medium – Perfect for smaller labels or secondary info.
      </p>
    </div>
  )
}
```

Note: Keep body text at a comfortable reading size (14px – 16px in typical desktop designs). Use smaller text only for labels, captions, and disclaimers.

## Best Practices

1. Maintain Hierarchy
Larger headings (Display 2xl, Display xl) should appear once per view or section. Use smaller displays or text headings for sub-sections.

2. Limited Weights
Don’t overuse bold or semibold in the same block of text. This helps preserve a clear emphasis hierarchy.

3. Responsive Adjustments
If you need finer control, consider making certain displays or text sizes slightly smaller on mobile devices by applying responsive classes like md:text-display-xl.

4. Accessibility
	-	Ensure sufficient color contrast with background elements.
	-	Use semantic HTML tags (`<h1>`, `<h2>`, `<p>`) for headings and paragraphs, even if you style them with custom classes.

## FAQ

<details>
  <summary><strong>How do I change the default font family?</strong></summary>
  Update <code>theme.extend.fontFamily</code> in your <code>tailwind.config.js</code> to set your custom typeface. Then ensure your entire application uses that typeface by default or selectively apply it via utility classes.
</details>


<details>
  <summary><strong>Can I rename "Display" and "Text" to something else?</strong></summary>
  Absolutely! Feel free to modify the naming scheme in your Tailwind config. Consistency within your team is what matters most.
</details>
