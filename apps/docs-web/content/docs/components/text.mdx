## Usage

Import the `Text` component and use the `variant` and `weight` props to apply the desired typographic style. You can also use the `as` prop for semantic HTML elements or `asChild` for composition.

```tsx
import { Text } from "@bank-kit/ui/components/text";

export default function MyComponent() {
  return (
    <div>
      {/* Use variant and weight */}
      <Text variant="displaylg" weight="bold">
        Account Overview
      </Text>

      {/* Default variant (textmd) and weight (regular) */}
      <Text>
        Welcome back! Here's a summary of your recent activity.
      </Text>

      {/* Use `as` prop for semantic elements */}
      <Text variant="textxl" weight="semibold" as="h2">
        Transactions
      </Text>

      {/* Use `className` for additional styles like color */}
      <Text variant="textsm" className="text-muted-foreground">
        Transfers may take up to 2 business days to settle.
      </Text>

      {/* Use `asChild` to style another component */}
      <Text variant="textsm" weight="medium" asChild>
        <a href="/learn-more">Learn More</a>
      </Text>
    </div>
  );
}
```

 ## Best Practices

To ensure consistency, accessibility, and maintainability when using the `<Text />` component, consider the following practices:

1.  **Prioritize Semantic HTML:**
    * Use the `as` prop (e.g., `as="h1"`, `as="label"`) or the `asChild` prop on semantic child elements (like `<button>`, `<a>`) to ensure your document structure is meaningful for assistive technologies and SEO.
    * Don't rely solely on visual styling (`variant="displaylg"`) to imply structure; use the correct underlying HTML tag.
    ```tsx
    // Good: Semantic H2 heading
    <Text variant="textxl" weight="semibold" as="h2">
      Section Title
    </Text>

    // Avoid: Looks like a heading, but is just a paragraph semantically
    <Text variant="textxl" weight="semibold">
      Section Title
    </Text>
    ```

2.  **Ensure Accessibility (A11y):**
    * **Color Contrast:** Always check that text has sufficient contrast against its background, meeting at least WCAG AA standards (4.5:1 for normal text, 3:1 for large text). Use browser developer tools or online checkers. Since color is often applied via `className`, be mindful of this during implementation.
    * **Readable Font Sizes:** Use appropriate `variant` values for readability based on context. `textmd` or `textlg` are generally suitable for body content. Avoid using very small variants (`textxs`) for critical information.

3.  **Maintain Consistency with Variants:**
    * Leverage the predefined `variant` and `weight` props as much as possible to maintain visual consistency across your application, adhering to the established design system.
    * Resist the urge to create one-off styles with `className` frequently. If a new style is needed repeatedly, consider adding a new variant to the `cva` configuration in `Text.tsx` and your `tailwind.config.js`.

4.  **Use Font Weights Purposefully:**
    * Apply `weight="semibold"` or `weight="bold"` intentionally for emphasis, headings, or calls to action.
    * Use `weight="regular"` for most body text to maximize readability. Avoid excessive use of bold text.

5.  **Align Numerical Data:**
    * When displaying numbers, especially financial data or figures in tables/lists where vertical alignment matters, apply the `tabular-nums` utility class via `className`. This ensures numbers take up uniform horizontal space.
    ```tsx
    <Text variant="textsm" className="tabular-nums">
      $ 1,234.56
    </Text>
    <Text variant="textsm" className="tabular-nums">
      $    78.90
    </Text>
    ```

6.  **Handle Custom Styling with `className`:**
    * For specific overrides like color, margins, or other utility-based styles not covered by `variant` or `weight`, use the `className` prop with Tailwind utilities.
    * This keeps the core component focused on typography structure and leverages Tailwind's utility-first approach.
    ```tsx
    <Text variant="textlg" className="text-primary mt-4">
      Important Note
    </Text>
    ```

7.  **Use `asChild` for Composition:**
    * Prefer `asChild` when you need to apply text styles directly to an interactive element (like a link or button) or another React component without introducing an extra wrapping `<span>` or `<div>`.
    * Use `as` primarily when you simply need to change the semantic HTML tag rendered by the `Text` component itself.
    ```tsx
    // Good: Styles applied directly to the anchor tag
    <Text variant="textsm" weight="medium" asChild>
      <a href="/terms">Terms of Service</a>
    </Text>

    // Less Ideal: Adds an extra span around the anchor
    <span className={textVariants({ variant: 'textsm', weight: 'medium' })}>
        <a href="/terms">Terms of Service</a>
    </span>
    ```
